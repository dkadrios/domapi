domapi.Imagepagecontrol

See also: Elm
See also: Component

Imagepagecontrol combines a Notebook and an Imagetabset component together to create a pagecontrol. Since it's based on the Imagetabset component, the visual look of the tabs is completely graphical.

 

Constructor Parameters unique to domapi.Imagepagecontrol

ParameterTypeDefaultDescription
orientationStringtop  Defines where the tabs are layed out. Accepted values are top, right, bottom and left.
overlapinteger0  Defines how much the tabs overlap each other.
paneManageStringvisibility  Defines how the control manages hiding/showing pages. Accepted values are visibility and display and correspond to their CSS equivalents.
You can also use Elm and Component constructor parameters when creating a domapi.Imagepagecontrol instance.

Adds a new page to the control. Returns a pointer to the new page.

Available "arg" Parameters:
ParameterTypeRequiredDefaultDescription
typeString DIV  The type of element the page is to be made out of.  
urlString   If the type is an IFRAME, this is the url for the page.  
delayLoadboolean false  If the type is an IFRAME, this parameter controls whether or not the page is loaded before the IFRAME is made visible by the tab being selected.  
refreshPageboolean false  When true and the type is IFRAME, this will cause the page to be refreshed each time the page is shown.
