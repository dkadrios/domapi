domapi.Outlookbar

See also: Elm
See also: Component

The Outlookbar component mimics the navigation panel found in Microsoft Outlook XP®. It is similar to Viewbar but offers more flexibility.

An Oulookbar consits of pages. Each page has a titlebar and a content pane. The titlebars can be grouped at the top or bottom of the control, or they can be 'justified' within the control. The content panes can contain and HTML or other DomAPI components. Only one content pane is visible at a time. The user makes a content pane visible by clicking on its titlebar.

Pages are managed using the addPage and removePage methods. The active page can be set using the setIndex method.

Constructor Parameters unique to domapi.Outlookbar

ParameterTypeDefaultDescription
valignStringjustfy  Affects how the tabs are layed out. Accepted values are top, bottom and justify. With top or bottom, all the tabs are kept together. With justify, the content pane is placed below the active tab.
You can also use Elm and Component constructor parameters when creating a domapi.Outlookbar instance.

Methods
addPage ( text, ref, id, enabled )

Adds a new page and tab to the control. The new page can be blank, or you can specify an existing element either by id or ref, to be used as the page.

Parameters:
ParameterTypeRequiredDefaultDescription
textStringY  The caption of the new tab.  
refHTMLElement   If present, this will cause the specified element to be used as the new page. If the element has already been added to the page, it will be removed and then reinserted into the control.  
idString   The id of an existing element. If present, that element will be used as the new page.  
enabledboolean true  Whether or not the new page is enabled.
