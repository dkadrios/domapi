domapi.Slider

See also: Elm
See also: Component

This is a fully "skinned" component. It utilizes the special "basin" routines of the drag extension to facilitate different stopping points for the slider's thumb control. If can also function as a smooth fader. Both the thumb and the slider background are images. The thumb itself is actually a Rollover component, and requires a 3 or 4 position image strip.

The component has two modes. These are basin and range. In basin mode, an array of coordinates is chosen in advance, These coordinates are the only places the user can drag the thumb to. The coordinates do not have to be in a straight line. The value of the control is the index in this basins array where the thumb currently rests.

When in range mode, the user can freely move the thumb within an area marked off by two coordinates. The values returned in the onchange event are percentages that the thumb has been moved in either x or y, making it an ideal mode for a fader control or similar.

Events
onchange ()

Fires when the thumb has been moved. The arguments passed depend on the mode.

basin
The event passes a single integer, which is the index in the basins array.

mode
The event passes two integers, which are the percentages of x and y within the range.

Properties
basinstype : Array

Used when mode is basin. The basins array is filled with two element arrays - one element for x, one for y. These define the basins or 'stopping points' within the control where the user is allowed to drag the thumb to.

Example:
slider1.basins = [
  [x1, y1],
  [x2, y2],
  [x3, y3],
];
