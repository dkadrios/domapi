Creating pages that use DomAPI

Author: Darin Kadrioski
Modified: 09/19/2004

All you need to do to a page to use DomAPI is include the "src/domapi.js" file on it using a script tag. The next section will get into this a bit more, but I'd like to take this time to get you aquainted with some basic concepts.

The library files come in one of two flavors. These are compressed and uncompressed. If you are using the standard library, then all files are compressed only. Compression drastically reduces the time it takes your users to download the code units at runtime. The compressed files have names that end with _c. For instance, the compressed version of domapi.js is domapi_c.js. One thing to keep in mind is that the version of domapi.js that is loaded determines which version of all the other files are used. In other words, if you load the uncompressed domapi.js unit, then all other units used on the page will be uncompressed as well; and vice-versa.
At runtime you can determine whether or not you are using compressed files by examining the domapi.compressed boolean property.

DomAPI is modular. Most all functionality is segregated into separate files. At runtime, DomAPI loads only the units it needs or that you tell it to. The idea is that the enduser's browser only needs to download exactly what it needs for your application and nothing more. This should help bring your pages up as quickly as possible. Also, if your user has their browser's cache turned on, then the Libraries files are only downloaded the first time they view your page.
DomAPI keeps track of which files it has loaded, and will never try to load the same file on the same page twice. But we'll get more into this in the section on domapi.loadUnit().
