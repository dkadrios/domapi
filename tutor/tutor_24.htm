Including other units

Author: Darin Kadrioski
Modified: 09/19/2004

Any unit that is not automatically load by domapi.js can be loaded by you using the domapi.loadUnit() method.

The following units are automatically loaded by domapi.js and never need to be loaded by you.

  • coreutil
  • css
  • elm
  • form_attach
  • theme
  • component
  • corecolor
  • lang
  • loaded

Any other extensions or components you require need to be loaded using domapi.loadUnit(). This needs to be done while the page is loading, before the onload event fires.

For example, if you wanted to use a Button component on your page: 

<script type="text/javascript" src="../scripts/domapi/src/domapi_c.js"></script>
<script type="text/javascript">
domapi.loadUnit("button");
onload = function(){
  myButton = Button();
};
</script>

Observe that the filename provided to domapi.loadUnit() is all lowercase and that there is no extension or path. DomAPI takes care of all this for you. If you used the compressed version of domapi.js, it will also take care of using the compressed button unit for you too, you would not add the '_c' part to the filename.

Dependancies

Some units are dependant on others, but they will always load the files they need by themselves, you don't have to worry yourself with that. For instance, the Listgrid component requires the dataset and customdrag units and will load them if they have not been loaded yet. DomAPI keeps tracks of which units have been loaded and will never load the same unit twice. You therefore do not have to worry about whether or not a unit has already been loaded before you make a call to domapi.loadUnit(). If you do ever need to know if a unit has been loaded though, you can use the unitLoaded method.
